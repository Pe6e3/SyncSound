# SyncSound

Два сервиса в одном репозитории:

- `client` — Vue 3 + TypeScript (Vite)
- `backend` — ASP.NET 8 Minimal API
- `nginx` — внешний вход, TLS и проксирование API

## Что реализовано

- API `GET /api/time` возвращает текущее время сервера в секундах Unix
- Клиент вызывает API каждые 5 секунд
- На главной странице отображается серверное время
- `nginx` отдает клиент и проксирует `/api/*` в backend

## Запуск

Требуется Docker с Compose.

```bash
docker compose up --build -d
```

После старта приложение доступно на порту **80** по HTTP (удобно в LAN без сертификатов):

- `http://<IP_сервера>/`

Остановка:

```bash
docker compose down
```

## HTTPS и Let's Encrypt (certbot)

Доступ к **микрофону** в браузере для калибровки возможен только в **безопасном контексте** (HTTPS или localhost). На сервере с реальным доменом настройте TLS.

### Подготовка

1. Домен должен указывать **A/AAAA-записью** на ваш сервер.
2. На сервере открыты порты **80** и **443** снаружи (для ACME http-01 и для HTTPS).
3. Скопируйте `.env.example` в `.env` и задайте `DOMAIN` (как в браузере) и `EMAIL` для ACME.

### Первичная выдача сертификата

На сервере в каталоге репозитория:

```bash
cp .env.example .env
# отредактируйте DOMAIN и EMAIL
chmod +x scripts/init-letsencrypt.sh
./scripts/init-letsencrypt.sh
```

Скрипт поднимает nginx, кратковременно подставляет самоподписанный сертификат, запрашивает настоящий сертификат **certbot** в режиме **webroot** и перезапускает nginx. Контейнер **certbot** в фоне раз в 12 часов выполняет `certbot renew`.

Тестовые сертификаты Let's Encrypt: в `.env` добавьте `STAGING=1` перед запуском скрипта (не для продакшена).

### Деплой через GitHub Actions

В корне репозитория на сервере создайте файл **`.env`** с теми же `DOMAIN` и `EMAIL` (в репозиторий `.env` не коммитится). Workflow перед `docker compose up` подхватывает переменные из `.env`, чтобы nginx знал имя хоста и при наличии каталога `certbot/conf/live/<DOMAIN>/` включал HTTPS.

После первого получения сертификата достаточно обычного `git pull` и деплоя; продление выполняет контейнер certbot.

По истечении срока действия сертификата файлы на диске обновляет `certbot renew`. Чтобы nginx подхватил новые PEM без перезапуска контейнера, на сервере можно выполнить:

```bash
docker compose exec nginx nginx -s reload
```

Имеет смысл вызывать это из cron раз в несколько дней после renew или после обновления проекта.
